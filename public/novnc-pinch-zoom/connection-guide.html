noVNC WebSocket 연결 문제

HTTPS 환경에서 WebSocket 연결 이슈 해결 가이드

문제: HTTPS 사이트에서 비보안 WebSocket (ws://) 연결이 차단됩니다.
원인: 브라우저의 Mixed Content 정책으로 인해 HTTPS 페이지에서 HTTP WebSocket 연결이 불가능합니다.

해결 방법 1: VNC 서버에서 WSS 지원 활성화

가장 권장되는 방법입니다. VNC 서버 설정에서 SSL/TLS를 활성화하세요.

websockify 프록시 사용 예시:

# SSL 인증서와 함께 websockify 실행 websockify --ssl-only --cert=server.crt --key=server.key 6080 localhost:5900

noVNC 연결 URL:

https://your-domain.com/vnc.html?host=112.214.237.240&port=6080&encrypt=1

해결 방법 2: 별도 WebSocket 프록시 서버 구축

WSS를 지원하는 별도의 프록시 서버를 구축합니다.

Node.js 프록시 서버 예시:

const WebSocket = require('ws'); const https = require('https'); // WSS 서버 생성 (SSL 인증서 필요) const wss = new WebSocket.Server({ port: 6080, // SSL 설정 추가 }); wss.on('connection', (ws) => { // VNC 서버로 프록시 const vncWs = new WebSocket('ws://112.214.237.240:5901/websockify'); ws.on('message', (msg) => vncWs.send(msg)); vncWs.on('message', (msg) => ws.send(msg)); });

해결 방법 3: 로컬 개발 환경 사용

HTTP 환경에서는 WebSocket 연결이 정상 작동합니다.

# 로컬 서버 실행 npm run dev # 브라우저에서 접속 http://localhost:4173/novnc-pinch-zoom/iframe-example.html
참고: 현재 VNC 서버 (112.214.237.240:5901)가 WSS를 지원하지 않는 것으로 보입니다. 서버 관리자에게 WSS 지원 활성화를 요청하거나, 위의 해결 방법 중 하나를 선택하세요.
